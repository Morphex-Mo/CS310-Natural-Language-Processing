# README 维护指南

本文档说明如何定期更新README文件，以保持内容的准确性和完整性。

## 📝 定期更新检查清单

### 每次添加新内容时

- [ ] 更新课程内容部分（作业/实验/讲义）
- [ ] 更新目录结构（如有变化）
- [ ] 更新技术栈（如引入新工具或库）
- [ ] 在更新日志中添加新条目
- [ ] 更新"Last Updated"日期

### 每周检查

- [ ] 检查所有文件路径是否正确
- [ ] 验证所有链接是否有效
- [ ] 确保代码示例可以正常运行
- [ ] 更新环境配置说明（如有依赖变更）

### 学期末检查

- [ ] 完善所有作业和实验的描述
- [ ] 添加课程总结
- [ ] 补充学习心得和建议
- [ ] 整理参考资源

## 🔄 快速更新模板

### 添加新作业

在"作业 (Coding Assignments)"部分添加：

```markdown
- **A[编号]**: [作业名称]
  - [作业简要描述]
  - 数据集：[数据文件列表]
```

### 添加新实验

在"实验课 (Lab)"部分添加：

```markdown
- **Lab [编号]**: [实验名称]
  - [实验要点1]
  - [实验要点2]
  - [使用的数据或工具]
```

### 添加新讲义

在"课程讲义 (Lecture)"部分添加具体讲义标题和主题。

### 更新日志格式

```markdown
- **YYYY-MM-DD**: [更新内容简述]
```

## 💡 建议的更新时机

1. **完成新的Lab后立即更新** - 趁记忆犹新，记录学习要点
2. **提交作业前更新** - 总结作业中的关键实现
3. **期中/期末考试后** - 补充重要知识点和复习建议
4. **每周日晚** - 定期回顾本周学习内容

## 🤖 自动化建议

考虑在每次Git提交时检查README是否需要更新：

```bash
# 查看最近的文件变更
git status

# 如果有新增的notebook或文档，记得更新README
```

## 📋 维护责任

- 保持README内容与实际文件结构一致
- 确保学术诚信声明清晰明确
- 及时记录学习过程中的重要发现
- 为后续学习者提供有价值的参考信息

---

**记住**：一个好的README是项目的门面，也是未来自己回顾学习的最佳入口！
